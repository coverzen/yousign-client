# Laravel Package Setup Guide

Standard setup for Laravel packages in our development team.

---

## Quick Checklist

**Initial Setup:**

- [ ] Request required config files from user (see "Required Configuration Files")
- [ ] Create project structure with versioned namespaces
- [ ] Set up `composer.json` with correct dependencies
- [ ] Create `PackageServiceProvider` and config file
- [ ] Set up git configuration (`.gitignore`, etc.)

**Before First Release:**

- [ ] All code follows coding standards (see php-agents.md)
- [ ] Run `composer analyse` (all tools passing)
- [ ] Run `composer test` (all tests passing)
- [ ] Create comprehensive README

**Directory Structure:**

- [ ] `src/` organized by standard Laravel application hierarchy and with `v1/` versioning
- [ ] `tests/Unit` mirrors `src/` structure
- [ ] `database/` has factories, migrations, seeders and constants, organized appropriately in folders
- [ ] `config/` has package configuration
- [ ] `routes/` has API routes

## Versioning Strategy

All code organized under versioned namespaces (e.g., `v1/`, `v2/`) to allow API changes without breaking existing
implementations. Apply versioning to: Enums, Events, Exceptions, Facades, Fakes, Http, Libs, Structs.

## Composer Configuration

**Key sections:**

- **require**: PHP 8.2+ or 8.3+
- **require-dev**: PHPStan, Psalm, PHP-CS-Fixer, CodeSniffer, Pest, etc.
- **autoload**: PSR-4 namespaces for `src/`, `database/factories/`, `database/constants/`
- **autoload-dev**: PSR-4 for `tests/`
- **extra.laravel**: Service providers and facades
- **scripts**: Composer commands for analysis and testing

**Essential composer scripts:**

- `composer analyse` - Run all quality checks
- `composer test` - Run tests with Pest
- `composer php-cs-fixer-fix` - Auto-fix code style
- Individual tools: `phpstan`, `psalm`, `phpcs`, `phpmd`, `phpmnd`

## Required Configuration Files

**MUST request from user before setup.** Do not generate these yourself:

**Code Quality:**

- `.php-cs-fixer.php` - PHP CS Fixer configuration
- `phpstan.neon` - PHPStan configuration
- `phpcs.xml` - PHP CodeSniffer configuration
- `psalm.xml` - Psalm configuration
- `phpmd.xml` - PHP Mess Detector configuration
- `phpinsights.php` - PHP Insights configuration

**Testing & CI/CD:**

- `phpunit.xml` - PHPUnit/Pest configuration
- `.gitlab-ci.yml` - GitLab CI/CD pipeline

User should supply based on team standards.

### Test Structure

- Use **Pest PHP** as testing framework
- Mirror `src/` directory structure in `tests/`
- Each test directory has corresponding `TestCase.php`
- Test files suffixed with `Test.php`
- **Always run tests with**: `ddev exec ./vendor/bin/pest`
- Refer to php-agents.md for testing standards

## Core Package Files

### Service Provider

Create `src/PackageServiceProvider.php` extending `ServiceProvider`:

```php
<?php declare(strict_types=1);

namespace Organization\Components\PackageName;

use Illuminate\Support\ServiceProvider;

final class PackageServiceProvider extends ServiceProvider
{
    public const CONFIG_KEY = 'package-name';

    public function boot(): void
    {
        $this->loadRoutesFrom(__DIR__ . '/../routes/api.php');

        if ($this->app->runningInConsole()) {
            $this->publishes([
                __DIR__ . '/../config/' . self::CONFIG_KEY . '.php'
                    => config_path(self::CONFIG_KEY . '.php'),
            ], 'config');
        }
    }

    public function register(): void
    {
        $this->mergeConfigFrom(
            __DIR__ . '/../config/' . self::CONFIG_KEY . '.php',
            self::CONFIG_KEY
        );
    }
}
```

### Configuration File

Create `config/package-name.php`:

```php
<?php declare(strict_types=1);

return [
    'option_name' => env('PACKAGE_OPTION_NAME'),
];
```
